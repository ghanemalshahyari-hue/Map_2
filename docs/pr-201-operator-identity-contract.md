# PR-201 — Operator Identity Contract

**Type:** Documentation only  
**Status:** Proposed  
**Depends on:** PR-200 (Diagnostics Display with Explicit Safe Snapshot)  
**Blocks:** PR-202 (Operator Identity Type Guard), PR-203 (Operator Identity Diagnostics Preview), PR-204 (Final Checklist Preview with identity status)

---

## 1. Purpose

Operator identity is a prerequisite for any future Gate 7 / Step 2 controlled apply confirmation.

The identity must identify the human operator who explicitly confirms the action. It is not a technical token derived from the machine state — it is a declaration of responsible human intent. For this reason:

- Operator identity must not be guessed, inferred, or generated by AI.
- Operator identity must not be auto-filled from any passive or hidden source.
- Operator identity must not fall back to a placeholder without explicit operator action.
- Until a real identity is available, the final confirmation preview remains **Not ready** and no apply path is unlocked.

The contract in this document governs how operator identity will be sourced, shaped, validated, and injected in future PRs. No implementation is introduced here.

---

## 2. Allowed and Forbidden Identity Sources

### Allowed future sources

| Source | `source` field value | Notes |
|---|---|---|
| Explicit operator input — the operator types or selects their identity directly | `"explicit_input"` | Highest trust. Requires a deliberate action by the human operator. |
| Authenticated local session identity, if a session module is approved in future | `"authenticated_session"` | Acceptable only if the session was created by an explicit, auditable login action, not by passive state. |
| Explicit test harness identity for diagnostics and testing | `"test_harness"` | Acceptable only in diagnostics. Must block Gate 7 live apply. `verified: false` required for live flow. |
| Future role/permission module, once approved by security review | `"role_module"` (future) | Not defined in this PR. Requires a separate contract. |

### Forbidden sources

| Forbidden pattern | Reason |
|---|---|
| AI-generated `operatorId` | Not a human declaration. Must not represent a real operator. |
| Random UUID or placeholder used for live apply | Not traceable to a specific human action. |
| Browser fingerprint (user agent, canvas, IP, etc.) | Passive, not declared, not auditable. |
| `localStorage` or `sessionStorage` identity | Persisted without explicit operator action at time of apply. May be stale. |
| Hidden default identity (`"operator"`, `"admin"`, `"user"`) without explicit source declaration | Masks the absence of real identity. |
| Anonymous live apply (empty or null `operatorId`) | No accountability. Must hard-block at Gate 7. |
| Auto-filled identity from any event, load hook, or timer | Not an explicit operator action. |
| Inherited identity from a previous session without re-confirmation | Stale, not fresh-at-action. |

## 3. OperatorIdentity Shape

A valid operator identity object has the following shape:

```js
{
  operatorId:   string,        // non-empty; identifies the operator
  displayName:  string | null, // human-readable name; optional
  role:         string | null, // operator role; optional; not used for gate logic
  source:       string,        // one of the allowed source values (§2)
  capturedAt:   string,        // ISO 8601 timestamp of when identity was captured
  verified:     boolean,       // true only when identity has been actively confirmed
  permissions: {
    canApproveDryRun:            boolean,  // Gate 3 operator review
    canConfirmControlledApply:   boolean   // Gate 7 Step 2 confirmation
  },
  readOnly:            true,   // hard-locked; must not be false
  liveMutationAllowed: false   // hard-locked; must not be true
}
```

### Field notes

| Field | Rule |
|---|---|
| `operatorId` | Non-empty string. No whitespace-only values. Must not be `"auto"`, `"default"`, `"placeholder"`, `"diagnostics-preview"`, or any reserved diagnostics label. |
| `displayName` | Optional. May be `null`. Not used for gate logic — display only. |
| `role` | Optional. May be `null`. Not used for gate logic in this contract. Future role-permission module may use it. |
| `source` | Required. Must match one of the allowed values in §2. Unknown or absent source is a hard block. |
| `capturedAt` | Non-empty ISO 8601 string. Must be set at the moment of capture, not deferred or backdated. |
| `verified` | `true` only when the identity has been actively confirmed by the operator in the current session. `false` is acceptable for test harness use, but blocks Gate 7 live apply. |
| `permissions.canConfirmControlledApply` | Must be `true` for Gate 7 Step 2. Absence or `false` is a hard block. |
| `readOnly` | Must be exactly `true`. |
| `liveMutationAllowed` | Must be exactly `false`. |

---

## 4. Validation Rules

An operator identity is valid for **real Gate 7 use** only when all of the following are true:

1. `operatorId` is a non-empty string and is not a reserved diagnostics label.
2. `source` is a known, allowed value (§2) and is not `"test_harness"`.
3. `capturedAt` is a non-empty string.
4. `verified === true`.
5. `permissions.canConfirmControlledApply === true`.
6. `readOnly === true`.
7. `liveMutationAllowed === false`.
8. No unsafe fields are present (see §7 for full list).

An operator identity with `source: "test_harness"` is valid **for diagnostics only**. It must never unlock Gate 7 live apply.

---

## 5. Diagnostics Behavior

Until a real operator identity is implemented:

- The final confirmation preview (`#sw-conf-section`) continues to display with `operatorId` row showing **"Unavailable — diagnostics preview"**.
- `buildApplyConfirmation` uses the synthetic context `{ operatorId: 'diagnostics-preview' }` as before (PR-193). This is clearly labelled as a diagnostics placeholder.
- The synthetic placeholder keeps `step2Complete: false` hard-locked — it does not unlock live apply.
- The final confirmation status row shows **"Not ready — apply candidate not ready"** (or "Preview only" if the candidate passes) — neither value implies that real identity is present.
- Adding a `test_harness` identity to `context.operatorIdentity` in a future PR must **not** change the final confirmation status from "Not ready" to any state that implies live apply is permitted.

---

## 6. Future Injection Path

Operator identity, once implemented, must be passed through the diagnostics context only — not read from globals, not cached in module scope, not pulled from storage.

The intended injection pattern is:

```js
paintStagingReadinessCard(context)
// where context may carry:
// {
//   liveUnitsSnapshot:  <LiveUnitsSnapshot>,   // PR-200
//   operatorIdentity:   <OperatorIdentity>      // future
// }
```

### Rules for context injection

| Rule | Detail |
|---|---|
| `context.operatorIdentity` is optional | Absence is handled gracefully — diagnostics fall back to "Unavailable" state. |
| Validate before use | Future paint code must call `isOperatorIdentitySafe(context.operatorIdentity)` before using any field. |
| No global operator identity | `window._operatorIdentity`, module-scope cache, or any persistent reference is forbidden. |
| No hidden cache | The identity must not be stored between paint calls. |
| No `localStorage` / `sessionStorage` lookup | The paint function must not read storage. |
| No fallback identity | If `context.operatorIdentity` is absent or invalid, the function must show "Unavailable" — it must not substitute a default. |
| No backend lookup | The paint function must not make any fetch or XHR call to obtain identity. |

## 7. Hard Blocks

Future controlled apply **must block entirely** if any of the following is true at the time of Gate 7 Step 2:

| Hard block condition | Code reference (future) |
|---|---|
| `operatorId` is missing, empty, or whitespace-only | `isOperatorIdentitySafe` → PR-202 |
| `source` is unknown, absent, or `"test_harness"` in a live flow | `isOperatorIdentitySafe` → PR-202 |
| `verified !== true` | `isOperatorIdentitySafe` → PR-202 |
| `permissions.canConfirmControlledApply !== true` | `isOperatorIdentitySafe` → PR-202 |
| Identity came from a placeholder or diagnostics source (`'diagnostics-preview'`, `'test_harness'`) | `buildApplyConfirmation` → PR-192 + future gate |
| Identity was auto-generated or derived without explicit operator action | `isOperatorIdentitySafe` source check |
| `readOnly !== true` | `isOperatorIdentitySafe` → PR-202 |
| `liveMutationAllowed !== false` | `isOperatorIdentitySafe` → PR-202 |
| Any unsafe field present on the identity object | Unsafe field list: `applyNow`, `commitNow`, `executeNow`, `mutateUnits`, `mutateMap`, `mutateLines`, `mutateScenario`, `backendCommit`, `autoApply`, `liveApply`, `autoConfirm`, `skipGate`, `forceApply` |
| Identity was cached from a prior session without re-confirmation | Staleness check — future contract |
